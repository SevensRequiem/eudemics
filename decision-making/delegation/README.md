# Delegation

How governance authority is structured and distributed.

## Federated Node System

No central authority. The government is a network of nodes spread across territory.

### Node Structure

- Territory divided into nodes, geography-aware: dense cities have several small nodes, sparse regions have fewer large nodes
- States/countries remain as geographic boundaries, nodes overlay onto them
- Each node is self-governing within the model's constraints
- All node officials are competency-verified to the same standard
- Nodes retain equal authority - no "capital" node or supreme council

### Dynamic Expansion

Nodes expand as civilization does:
- Space currently needs one node
- Moon colonization creates a moon node. As lunar infrastructure grows, the moon node subdivides.
- Same for ocean settlements, arctic stations, Mars
- Barren/frontier regions start with large single nodes that compartmentalize as population and infrastructure justify it

### Rotation and Succession

- Node members serve 2-year terms every 5 years (rotating)
- Candidates are randomly pre-picked and rotated every 6 months as backups
- If a member dies or becomes incapacitated, the pre-picked candidate steps in immediately
- No succession crisis is possible because the replacement is always already identified
- Random selection from the competency-verified pool prevents dynasties and political maneuvering

### Representation Caps

No religious, ideological, or cultural group exceeds 30-40% of any governing body. This includes atheism (also unproven, gets the same treatment).

The logic: more than 50% of any group inevitably biases decisions toward their worldview, even unintentionally. The cap forces coalition and compromise. No single perspective dominates policy.

### Military Integration

Nodes are uniform in structure but have specialized offices: governance, military, engineering, medical, etc. An engineer and a strategist sit in the same node but different offices. The engineer has technical insight the strategist lacks, and vice versa. Both contribute.

Military resource assignment is situation-appropriate:
- Cyber attack: nodes with cybersecurity resources handle it, not farming nodes
- Local conflict: neighboring nodes contribute
- Ballistic/asteroid threat: nodes with relevant defense infrastructure activate
- Each node decides whether contributing to a distant conflict would detriment its own efficiency

All military personnel are extremely vetted, with higher competency requirements than civilian governance for sensitive roles. See [[security/defense]].

### War Resilience

The federated structure means losing a node does not collapse the system. All data is replicated across nodes (see [[security/resilience]]). Any node can functionally replace any other because all officials meet the same competency standard. There is no single point of failure.
